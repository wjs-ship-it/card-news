# 🔐 Security Checklist

## ✅ 현재 구현된 보안 사항

- [x] API 키는 환경변수 또는 로컬 설정 파일로만 관리
- [x] .env 파일은 .gitignore에 포함
- [x] 설정 파일은 ~/.card-news/에 저장 (사용자 홈 폴더 - 비공개)
- [x] 민감한 정보는 코드에 하드코딩 안 함
- [x] GitHub에 올릴 때 보안 정보 제외
- [x] 에러 메시지에서 API 키 노출 방지
- [x] package-lock.json은 의존성 체크를 위해 포함

## 🔍 Phase 2에서 추가할 것

- [ ] 배치 파일 입력 검증 (경로 트래버설 공격 방지)
- [ ] 임시 파일 정리 (생성 후 삭제)
- [ ] ZIP 파일 생성 시 경로 검증
- [ ] 파일 시스템 접근 권한 확인
- [ ] 사용자 입력 sanitization

## 🚫 절대 하면 안 되는 것

❌ API 키를 소스 코드에 하드코딩
❌ 민감한 정보를 주석에 포함
❌ 설정을 JSON에 평문으로 저장 (이미 우리는 사용자 입력으로만 함)
❌ 에러 로그에 전체 스택 트레이스 + 민감 정보 노출
❌ 패스워드/토큰을 CLI 인자로 받기 (환경변수 사용)
❌ 임시 파일을 정리 안 하고 남겨두기

## 📋 코드 리뷰 체크리스트

- [ ] `process.env`로 민감한 값 읽기
- [ ] 에러 메시지가 사용자 친화적인지 (민감 정보 없이)
- [ ] 파일 경로가 상대 경로로 시작하거나 사용자가 제어할 수 있는지 확인
- [ ] 임시 파일이 finally 블록이나 안전한 방식으로 정리되는지
- [ ] 사용자 입력이 검증되는지
